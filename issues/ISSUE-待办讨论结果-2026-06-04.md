## 一、确定要做的事项（5 项）

### 1. 断点续跑

**问题**: 任务执行中断（浏览器关闭/网络断开）后，in_progress 状态的任务卡住，既不是 pending（不会被执行）也不是 completed。再次打开会话时无法继续执行。

**当前行为**: `execution.ts:66` 只执行 `status === 'pending'` 的任务，in_progress 的任务被跳过。

**方案**: 加载会话时，自动把 in_progress 任务重置为 pending。已完成的任务天然被跳过，所以只需重置卡住的任务即可从断点继续。

**改动点**:
- `src/app/api/sessions/[id]/route.ts` — GET 会话详情时，顺带把 in_progress 任务改为 pending
- 或新建恢复 API，前端打开会话时调用

**改动量**: ~15 行

---

### 2. RECOVER-001: 任务恢复提示 UI

**依赖**: 断点续跑功能完成后。

**问题**: 用户打开会话时不知道上次有未完成的任务。

**方案**: 打开会话时检查是否有 in_progress（即将被重置为 pending）的任务，弹提示"上次有未完成的任务，是否继续执行？"

**改动点**:
- 前端会话加载逻辑 + 确认弹窗
- 点"继续"触发执行，点"跳过"忽略

---

### 3. DIFF-001: Diff Accept 前文件修改检测

**问题**: 用户在 AgentHub 看 diff 点 Accept 时，如果同时在编辑器里改了同一个文件，Agent 的产出会覆盖用户的手动修改，无任何提示。

**方案**:
- diff 展示时记录文件内容 hash（md5）
- Accept 时重新计算 hash 对比
- 不一致则弹确认框"文件已被修改，是否覆盖？"

**改动点**:
- `src/app/api/sessions/[id]/files/accept/route.ts` — Accept 前校验 hash
- `src/components/code-diff.tsx` — 展示 diff 时传 hash 给前端
- 前端确认弹窗

**改动量**: ~50 行

---

### 4. FAIL-007: 全链路 trace

**问题**: 任务失败后只知道"失败了"，不知道跑了多久、重试了几次、每次报什么错、Orchestrator 为什么选了这个 Agent。SSE 是瞬时的，关掉页面就没了。

**方案**: Task 表加 `trace` JSON 字段，记录结构化执行轨迹：

```json
[
  { "ts": "2026-06-04T09:00:01Z", "event": "start", "agent": "后端工程师", "platform": "claude-code" },
  { "ts": "2026-06-04T09:00:15Z", "event": "error", "message": "API timeout" },
  { "ts": "2026-06-04T09:00:16Z", "event": "retry", "attempt": 1 },
  { "ts": "2026-06-04T09:00:30Z", "event": "success", "duration_ms": 29000 }
]
```

**改动点**:
- `prisma/schema.prisma` — Task 表加 `trace String @default("[]")`
- `src/lib/orchestrator/index.ts` — `executeSingleAgent` / `executeTaskBatch` 各关键节点写入 trace
- `src/lib/services/execution.ts` — 执行流程中记录 trace
- 前端 Task 卡片可选展示 trace 详情

**改动量**: ~100 行

---

### 5. BUG-003: 群聊创建 checkbox 双重 toggle

**问题**: 群聊创建向导第二步选 Agent 时，直接点 checkbox 框不生效，必须点整行文字才能选中。

**根因**: `src/components/create-group-dialog.tsx:275-287`，checkbox 的 onChange 和父 div 的 onClick 都调用 `toggleAgent()`。点 checkbox 时触发两次：onChange 选中 → 冒泡到父 div onClick 取消，净效果为 0。

**修复**: 去掉父 div 的 onClick，只保留 checkbox 的 onChange；或者在 checkbox 上加 `e.stopPropagation()`。

**改动量**: ~3 行
